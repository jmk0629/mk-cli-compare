# 2026-06-22 — AI 자동 심판 (LLM-as-judge)

## 아이디어
- 사람 블라인드 투표는 표가 쌓여야 의미가 생긴다. **AI 심판**은 비교 1건만으로 즉시 객관 평가를 준다.
- 3 응답을 A/B/C 로 가린 채 심판 CLI 에 보내 **순위·점수(0~10)·이유**를 받고, 사람 투표와 나란히 비교.

## 만든 것
- **DB V6** `judge_verdict`: 평결 저장(verdict_json + winner 컬럼 승격).
- **백엔드**
  - `JudgeService`: ok 응답에 A/B/C 라벨 부여(정체 가림) → 심판 프롬프트(JSON 강제) → CLI 호출 →
    첫 `{`~마지막 `}` 추출 후 트리 파싱 → 라벨→providerId 환원 → 최고점 winner. 실패는 error 평결로 저장.
  - `POST /api/comparisons/{id}/judge {provider?}` (옵트인, CLI 1회), `GET …/judge`(최근 평결, 없으면 204).
  - 심판 provider 기본 claude(요청 시 변경). 자기편향은 블라인드로 완화 + UI 고지.
- **프론트** `JudgePanel`(상세 페이지): 평결 자동 로드, "AI 심판에게 평가받기"/"다시 심판" 버튼,
  winner 👑 + 점수 막대 + 이유 + 총평 + 심판 고지.

## 검증 (라이브)
- FizzBuzz 비교 심판: winner=claude(9.2 "호출 포함·명료"), gemini 8.5("호출 없어 미실행"), codex 8.0("트릭이나 가독성↓").
  블라인드 라벨 매핑·점수·이유 정확. 상세 페이지 렌더 확인(스크린샷).
- SmokeTest 갱신(프리셋 23), 전체 테스트 통과.

## 확장 여지
- 3-심판 패널(각 CLI가 심판, 다수결)로 자기편향 추가 완화.
- 심판 승률을 리더보드에 "AI 평가 기준" 별도 집계.
